Virtual environment

In this chapter, we will explore how to set up a Python development environment using Conda, a powerful package and environment management system. Conda allows us to create isolated environments for different projects, manage dependencies seamlessly, and ensure reproducibility across different computing environments.

Prerequisites

Before we begin, make sure you have the following:

  • Conda Installed: If you haven’t installed Conda yet, visit the Conda website and download the installer suitable for your operating system.
  • Basic Knowledge of Terminal/Command Line: Familiarity with basic terminal or command line operations will be helpful.

Step 1: Installing Conda

  1. Download Anaconda or Miniconda:
    • Anaconda: Includes Conda, Python, and a large number of packages commonly used in data science and scientific computing.
    • Miniconda: A minimal installer that includes only Conda, Python, and essential packages.
  2. Install Anaconda or Miniconda: Follow the installation instructions provided on the Conda website for your operating system. Once installed, Conda should be available in your terminal or command prompt.

Step 2: Creating a Conda Environment

  1. Open Terminal or Command Prompt: Open your terminal application (e.g., Terminal on macOS, Command Prompt on Windows).

  2. Create a New Conda Environment: To create a new environment named myenv with Python 3.8 (adjust the version as needed), use the following command:

    conda create --name myenv python=3.10

    Replace myenv with your preferred environment name and python=3.10 with your desired Python version.

  3. Activate the Environment: After creating the environment, activate it using the following command:

    conda activate myenv

    Your terminal prompt should now show the active environment name (myenv).

Step 3: Installing Python Packages

  1. Using Conda: Install packages from the Conda repository. For example:

    conda install numpy pandas matplotlib

    Replace numpy, pandas, and matplotlib with any packages you need for your project.

  2. Using pip: Install packages from PyPI (Python Package Index). For example:

    pip install requests flask

    Replace requests and flask with any packages not available in the Conda repository.

Step 4: Optional - Installing Jupyter Notebook

If you plan to use Jupyter Notebook for interactive development:

  1. Install Jupyter Notebook: Use the following command to install Jupyter Notebook within your Conda environment:

    conda install jupyter

Step 5: Deactivating and Removing Environments

  1. Deactivate the Environment: When you’re done working in your environment, deactivate it using:

    conda deactivate

  2. Removing the Environment (Optional): If you want to remove a Conda environment entirely:

    conda remove --name myenv --all

    Replace myenv with the name of your environment.

Conclusion

Setting up a Python development environment using Conda provides a robust and flexible approach to managing dependencies and isolating project environments. It ensures that your projects are reproducible across different systems and simplifies package management.
